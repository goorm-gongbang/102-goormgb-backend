## Claude 행동 규칙

### 커밋 관련
- **커밋을 자동으로 생성하지 않는다**
- 사용자가 요청 시 커밋 메시지 **내용 추천만** 진행한다
- 해당 브랜치의 커밋 이력, 작업 내용, diff를 참고하여 추천한다
- 커밋 메시지는 아래 규칙을 따른다:
  - 타입은 영어 소문자
  - 제목은 50자 이내 한글
  - 제목은 명령문 형태, 과거형 금지
  - 제목 끝에 마침표 없음

### PR 관련
- **PR을 자동으로 생성하지 않는다**
- `pr 내용 추천` 요청 시 아래 포맷에 맞춰 추천한다
- 해당 브랜치의 커밋 이력과 diff를 분석하여 작성한다

### PR 추천 포맷

```markdown
제목: [type](scope): subject

## 🔧 작업 내용
- 무엇을 개발했는지
- 어떤 문제를 해결했는지
- 왜 이런 방식으로 구현했는지

## 🧩 구현 상세 (선택)
- 핵심 로직 설명
- 설계 / 구조 / 알고리즘 / 모델 선택 이유
- 트레이드오프 또는 고민했던 지점

### 📌 관련 Jira Issue
- GRGB-XX

## 🧪 테스트 방법 (선택)
- 테스트 대상
- Endpoint / 함수 / 스크립트
- 파라미터 및 체크 포인트

## ❗ 참고 사항
- 리뷰 시 유의할 점
- 후속 작업 예정
- 배포 시 주의 사항
```

### 타입 분류

| 타입 | 설명 |
| --- | --- |
| **feat** | 새로운 기능 추가 |
| **fix** | 버그 수정 |
| **build** | 빌드 관련 변경 |
| **chore** | 자잘한 변경 |
| **ci** | CI/CD 관련 설정 변경 |
| **docs** | 문서 수정 |
| **style** | 포맷팅 등 비기능적 수정 |
| **refactor** | 코드 리팩터링 |
| **test** | 테스트 코드 추가/수정 |
| **perf** | 성능 개선 |

## Issue 템플릿

### Bug / Code Fix Issue Template

> 재현 가능한 버그 혹은 코드 구조 개선이 필요한 경우
> labels: `bug`, `fix`

```markdown
제목: [bug] 내용 요약

## 🔥 문제 요약
- (버그 또는 코드 문제 한 줄 요약)

## 📍 상세 내용
- 어떤 동작 중 발생했는지?
- 로그/스크린샷/에러 메시지 (선택)

```java
[에러 로그 또는 관련 코드 블록]
```

## 🧪 재현 방법 (선택)
1. …
2. …
3. …

## 🧭 원인 분석 생각 (선택)
- (추정 원인 적기 — 명확하지 않아도 OK)

## 🔧 예상 해결 방향
- [ ] 수정 포인트
- [ ] 리팩토링 필요 부분
- [ ] 테스트 필요 여부

## 📌 연결 PR (선택)
- #PR-number (추후 작성)
```

### Quick Issue (사소한 변경·아이디어·TODO 기록용)

> 사소한 오류 / 개선 아이디어 / 작업 메모 기록용
> labels: `chore`, `minor`, `improvement`

```markdown
제목: [issue] 내용 간단 요약

## ✏ 작업 내용 / 하고 싶은 변경사항
- (짧게 한 줄로 — 아는 만큼만 써도 됨)

## 📝 상세 설명 (optional)
- (추가 설명 / 이유 / 링크 첨부 가능)
- ex) Soft delete 로직과 맞지 않는 조회 부분 확인 필요

## 📅 계획
- [ ] 나중에 보기
- [ ] 이번 Sprint 처리
- [ ] 지금 바로 수행

## 🔗 관련 코드/PR/커밋 (optional)
- 링크 또는 스크린샷 첨부
```
